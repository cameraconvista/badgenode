### 8.2 SOLUZIONE ROBUSTA: Standardizzazione PIN

```javascript
// Funzione utility per normalizzare PIN
function normalizzaPin(pin) {
  if (!pin) return null;
  
  // Estrae solo numeri
  const numericPin = pin.toString().replace(/[^0-9]/g, '');
  
  if (numericPin.length === 0) return null;
  
  return parseInt(numericPin);
}

// Uso nelle query
.eq("pin", normalizzaPin(pin))
```

### 8.3 SOLUZIONE PREVENTIVA: Validazione link

```javascript
// In utenti.html - validazione prima di generare link
function generaLinkStorico(utente) {

    return '#'; // Link disabilitato
  }
  return `storico.html?pin=${utente.pin}`;
}
```

### 10.1 ROOT CAUSE IDENTIFICATA

**PROBLEMA PRINCIPALE:**
Il PIN "04/06" contiene caratteri non numerici ("/") che causano parsing errato:
- `parseInt("04/06")` = `4`
- Se il dipendente ha PIN diverso da 4, la query non trova risultati
