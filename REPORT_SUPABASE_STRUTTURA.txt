## TABELLE DEL DATABASE

### 1. TABELLA: utenti
**Scopo**: Gestione anagrafica dipendenti
**Campi identificati dal codice**:
- id (PRIMARY KEY) - Identificativo unico
- pin (INTEGER) - PIN numerico per timbratura (1-2 cifre)
- nome (TEXT) - Nome dipendente
- cognome (TEXT) - Cognome dipendente  
- email (TEXT) - Email dipendente

**Operazioni CRUD**:
- SELECT: Recupero lista utenti, verifica PIN
- INSERT: Aggiunta nuovo dipendente
- UPDATE: Modifica dati dipendente
- DELETE: Eliminazione dipendente

**Vincoli**:
- PIN deve essere unico (errore 23505 gestito)
- Validazione PIN: 1-2 cifre numeriche
- Validazione email: formato base

### 2. TABELLA: timbrature
**Scopo**: Registrazione entrate/uscite dipendenti
**Campi identificati dal codice**:
- id (PRIMARY KEY) - Identificativo unico timbratura
- pin (INTEGER) - PIN dipendente (FK verso utenti.pin)
- nome (TEXT) - Nome dipendente (denormalizzato)
- cognome (TEXT) - Cognome dipendente (denormalizzato)
- tipo (TEXT) - Tipo timbratura: "entrata" | "uscita"
- data (DATE) - Data timbratura (formato ISO: YYYY-MM-DD)
- ore (TIME) - Orario timbratura (formato: HH:mm:ss)
- giornologico (DATE) - Data del giorno lavorativo

**Operazioni CRUD**:
- SELECT: Recupero storico, verifica ultima timbratura
- INSERT: Registrazione nuova timbratura
- UPDATE: Modifica timbrature esistenti (da storico.html)
- DELETE: Eliminazione timbrature (da storico.html)

**Query principali**:
1. Ultima timbratura per PIN:
   SELECT tipo FROM timbrature 
   WHERE pin = ? 
   ORDER BY data DESC, ore DESC 
   LIMIT 1

2. Storico per periodo:
   SELECT * FROM timbrature 
   WHERE pin = ? AND data BETWEEN ? AND ?
   ORDER BY data DESC, ore DESC
